# Development Plans

This folder contains structured development plans generated by the `@plan-generator` agent.

## Quick Start: Autonomous Mode

Run the entire workflow with a single command:

```
@plan-orchestrator Run workflow from {roadmap.md}
```

The orchestrator automatically runs all agents in sequence, handles failures with retries, and reports progress.

---

## Cascade Workflow

```
                                    ┌───────────────────┐
                                    │ @plan-orchestrator│ ← Single command
                                    │   (autonomous)    │
                                    └─────────┬─────────┘
                                              │ orchestrates
                                              ▼
┌─────────────────┐     ┌──────────────────┐     ┌──────────────┐     ┌────────────────┐     ┌─────────────────┐
│ @plan-generator │ ──▶ │ @plan-implementer│ ──▶ │ @plan-tester │ ──▶ │ @plan-validator│ ──▶ │ @plan-documenter│
│   Create Plan   │     │  Implement Code  │     │  Unit Tests  │     │   E2E Tests    │     │  Documentation  │
└─────────────────┘     └──────────────────┘     └──────────────┘     └────────────────┘     └─────────────────┘
                               │                        │                     │
                               ◀────── Rework ──────────┴─────────────────────┘
```

### Step-by-Step (Manual Mode)

1. **Generate**: `@plan-generator` + attach roadmap/notes
2. **Implement**: `@plan-implementer Start plan {name}`
3. **Unit Test**: `@plan-tester Test plan {name}`
4. **Rework** (if needed): `@plan-implementer` fixes failed steps
5. **E2E Test**: `@plan-validator Validate plan {name}`
6. **Document**: `@plan-documenter Document plan {name}`
7. **Complete**: Plan reaches `DOCUMENTED` status ✅

### Status Flow

```
Step Level:
NOT_STARTED → IN_PROGRESS → REVIEW → VERIFIED
                   ↑           │
                   └── FAILED_* ┘

Plan Level:
Draft → Ready for Implementation → VALIDATED → DOCUMENTED ✅
              ↑                         │
              └─────── ROLLBACK ────────┘
```

## File Naming

Plans are named: `PLAN-{feature-name}.md`

## How to Use

### Autonomous Mode (Recommended)

Let the orchestrator handle everything:

```
@plan-orchestrator Run workflow from docs/roadmaps/roadmap.md
```

Resume from a specific stage after manual fixes:

```
@plan-orchestrator Continue from tester for plan auth-system
```

The orchestrator will:
- Run all 5 agents in sequence
- Auto-retry failed stages (up to 3 times)
- Report progress after each stage
- Handle rework loops automatically

---

### Manual Mode

If you prefer step-by-step control:

### 1. Generate a Plan

Attach your roadmap, notes, or requirements and invoke the generator:

```
@plan-generator Create plan from attached roadmap
```

Or reference existing files:

```
@plan-generator Create plan based on docs/roadmaps/roadmap.md and docs/requirements.md
```

The agent will create `docs/plans/PLAN-{name}.md` with structured sections.

### 2. Implement the Plan

Start implementation from the beginning:

```
@plan-implementer Start plan {name}
```

Or continue from where you left off:

```
@plan-implementer Continue plan {name}
```

Check progress:

```
@plan-implementer Status
```

### 3. Run Unit Tests

After implementation, validate with unit tests and code quality checks:

```
@plan-tester Test plan {name}
```

Test specific steps:

```
@plan-tester Test step 2.1
```

If tests fail, the tester returns steps to the implementer:

```
@plan-implementer Continue plan {name}   # Fix failed steps
@plan-tester Test plan {name}            # Re-test
```

### 4. Run E2E Tests

Once all steps are `VERIFIED`, run end-to-end validation:

```
@plan-validator Validate plan {name}
```

The validator creates tests in `tests/e2e/{plan-name}/` and runs full workflow validation.

### 5. Document the Code

After validation passes, add documentation:

```
@plan-documenter Document plan {name}
```

Document specific files:

```
@plan-documenter Document file src/auth.py
```

### Quick Reference

| Mode | Command | Output |
|------|---------|--------|
| **Autonomous** | `@plan-orchestrator Run workflow from {source}` | `DOCUMENTED` ✅ |

| Stage | Command (Manual Mode) | Output Status |
|-------|---------|---------------|
| Generate | `@plan-generator Create plan from {source}` | `Draft` |
| Implement | `@plan-implementer Start plan {name}` | `REVIEW` |
| Unit Test | `@plan-tester Test plan {name}` | `VERIFIED` |
| E2E Test | `@plan-validator Validate plan {name}` | `VALIDATED` |
| Document | `@plan-documenter Document plan {name}` | `DOCUMENTED` ✅ |

### Handling Failures

When a step fails at any stage:

1. Check the plan for failure details and notes
2. Return to `@plan-implementer` to fix the issue
3. Re-run the failing stage
4. Continue the cascade

```
# Example: E2E test failed on step 2.1
@plan-implementer Fix step 2.1        # Fix the issue
@plan-tester Test step 2.1            # Re-run unit tests
@plan-validator Validate plan {name}  # Re-run E2E
```

## Output Artifacts

After a plan reaches `DOCUMENTED` status, the following artifacts are produced:

| Artifact | Location | Description |
|----------|----------|-------------|
| Plan Document | `docs/plans/PLAN-{name}.md` | Full plan with status flags and reports |
| E2E Tests | `tests/e2e/{plan-name}/` | End-to-end test suite |
| API Docs | `docs/api/` | Generated API documentation |
| **Roadmap Summary** | `docs/docs/readme_roadmap.md` | **Final documentation of all implemented features** |

### Roadmap Documentation

The `@plan-documenter` appends completed features to [docs/docs/readme_roadmap.md](docs/readme_roadmap.md) as a permanent record of implemented roadmap items. This file serves as the central reference for all finalized development work.
